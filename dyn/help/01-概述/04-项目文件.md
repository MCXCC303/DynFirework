# 项目文件

DynFirework 使用 `.dyn` 后缀的压缩归档文件（tar.gz 格式）保存项目。文件内部结构如下：

- `manifest.json`：项目元数据，包含项目名称、后端类型（`cb` 或 `df`）、目标 Minecraft 版本、BPM、`format_version` 字段以及各文件的 SHA-256 校验和。
- `elements/`：目录，每个元素以独立的 JSON 文件存储，按元素类型归类。
- `positions.json`：保存的预设位置坐标数据。
- `timeline_state.json`：时间线的 UI 状态，包括缩放级别和滚动偏移。
- `assets/`：嵌入的音乐等资源文件。

`format_version` 当前为 `"1.0"`（首个公开版本）。版本号用于后续版本的向后兼容性检测，导入旧格式文件时自动适配。

后端在项目创建时锁定，保存在 `manifest.json` 的 `backend` 字段中，之后无法更改。音乐文件直接嵌入归档内部，移动 `.dyn` 文件到其他设备不会丢失音频数据。
